# Shared Memory

Store durable, non-secret knowledge that is useful across projects and machines:

- engineering principles and terminology;
- stable workflow preferences;
- shared conventions that do not fit a single skill;
- curated reference maps with an owner and review date.

Keep this layer concise. A fact that is relevant only to one task, domain, or repository should live in a skill reference or a repository-local instruction file instead.

Never store credentials, personal data, temporary incident state, or host-specific paths here.
